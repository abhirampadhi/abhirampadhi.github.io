Introduction to Entity Framework Core

September 22, 2024

Entity Framework Core (EF Core) is a powerful Object-Relational Mapper (ORM) for .NET applications. It simplifies data access by allowing developers to work with databases using .NET objects rather than SQL queries. In this article, we’ll provide an introduction to EF Core, its features, and how to get started with it in your .NET applications.

What is Entity Framework Core?

EF Core is a lightweight and extensible version of Entity Framework, designed to work with .NET Core and .NET Framework applications. It enables developers to interact with databases using LINQ queries and provides features such as automatic migrations, change tracking, and lazy loading.

Key Features of EF Core

  • Code-First Approach: Allows developers to define their database schema using C# classes and automatically generate database tables based on these classes.
  • LINQ Queries: Enables querying and manipulating data using LINQ, providing a more intuitive and type-safe way to interact with the database.
  • Automatic Migrations: Automatically updates the database schema based on changes to the entity classes.
  • Change Tracking: Tracks changes to entities and handles updates to the database efficiently.

Getting Started with EF Core

Step 1: Install EF Core Packages


dotnet add package Microsoft.EntityFrameworkCore
dotnet add package Microsoft.EntityFrameworkCore.SqlServer
dotnet add package Microsoft.EntityFrameworkCore.Tools

Install the necessary EF Core packages using the .NET CLI to get started with EF Core in your project.

Step 2: Define Your Data Model


public class Blog
{
    public int BlogId { get; set; }
    public string Name { get; set; }
    public List Posts { get; set; }
}

public class Post
{
    public int PostId { get; set; }
    public string Title { get; set; }
    public string Content { get; set; }
    public int BlogId { get; set; }
    public Blog Blog { get; set; }
}

Define your entity classes, which represent the data model for your application. In this example, we have `Blog` and `Post` entities.

Step 3: Configure Your DbContext


public class BloggingContext : DbContext
{
    public DbSet Blogs { get; set; }
    public DbSet Posts { get; set; }

    protected override void OnConfiguring(DbContextOptionsBuilder optionsBuilder)
    {
        optionsBuilder.UseSqlServer("YourConnectionString");
    }
}

Create a `DbContext` class to manage the database connection and entity sets.

Step 4: Perform Migrations


dotnet ef migrations add InitialCreate
dotnet ef database update

Use the Entity Framework Core CLI to add migrations and update the database schema.

Conclusion

Entity Framework Core is a versatile ORM that simplifies database interactions in .NET applications. By leveraging EF Core, developers can write more maintainable and efficient data access code. Whether you’re building a new application or working on an existing one, EF Core is a valuable tool for managing data in .NET applications.
